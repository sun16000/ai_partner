## 三、遇到的问题

### 1. 只有一个if条件判断语句，为何能实现循环问答

> [!NOTE]
> 在普通的 Python 脚本中，代码从上到下执行一次就结束了。但在 Streamlit 应用中：
> 每次用户与界面交互（比如点击按钮、输入文本、选择下拉框等），整个脚本都会从上到下重新运行一遍。

**具体到这个例子：**

1. 第一次打开页面
   脚本从头执行到 `st.chat_input(...)`，此时用户还没有输入，`user_prompt` 为 `None`，所以 `if` 条件不成立，页面只显示一个输入框，等待用户输入。

2. 用户输入问题并提交
    Streamlit 检测到输入事件，重新运行整个脚本。这次 `user_prompt` 有了内容（用户输入的文字），`if` 条件成立，于是：
    - 显示用户的消息（`st.chat_message("user").write(...)`）
    - 调用 DeepSeek API
    - 显示助手的回复（`st.chat_message("assistant").write(...)`）

3. 脚本执行结束
    脚本执行结束，但页面仍然显示输入框，用户又可以输入下一个问题。

4. 循环问答
    输入新问题 → 再次重新运行脚本 → 再次进入 if 分支 → 再次问答。

### 2. if user_prompt 语句的必要性
**核心作用**：防止页面首次加载时，因为还没有用户输入而错误地去调用 AI 大模型。

🧠# 理解 Streamlit 的执行机制

Streamlit 应用不是像普通 Python 脚本那样只运行一次，而是：

> 每次用户与界面交互（包括首次打开页面），整个脚本都会从上到下重新执行一遍。

这意味着：

- 当用户第一次打开你的 AI 伴侣页面时，脚本开始执行。
- `st.chat_input("请输入你的问题")` 此时返回 `None`（因为用户还没有输入任何文字）。
- 如果没有 `if user_prompt:` 判断，脚本会继续往下执行：
  - `st.chat_message("user").write(user_prompt)` → 试图显示 `None`（可能显示为空，但不报错）
  - 创建 `OpenAI` 客户端
  - 调用 API，但 `messages` 中的 `user` 内容是 `None`
  - 最终导致 API 调用失败或返回异常结果。
